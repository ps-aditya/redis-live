# Product Principles

## Principle 1

Redis first.

Every feature must improve understanding of Redis.

## Principle 2

Observation over instruction.

The product should show users what happened rather than simply telling them.

## Principle 3

Experimentation over courses.

Users should be encouraged to test ideas and explore behavior.

## Principle 4

Cause and effect.

Every command should clearly show:

Command

↓

State Change

↓

Visualization

↓

Understanding

## Principle 5

Transparency.

Redis should never feel like a black box.

## Principle 6

Avoid unnecessary complexity.

If a feature does not improve understanding or observability, it should not be built.
